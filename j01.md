# Trying to Build My First Circuit in LTspice

For one of my engineering projects, I needed to create and simulate a simple circuit using LTspice. Since this was my first time using this software using a schematic rather than a netlist, my goal was to build a basic circuit with a voltage source and a resistor, then observe the voltage and current values.
# Initial Interaction
When I first opened LTspice, I was presented with a blank workspace and a toolbar filled with small, unclear icons.

There were no instructions or clear indication of how to begin. Based on my experience with other software, I assumed that clicking a plus icon would allow me to start creating a new workspace or file. After hovering over the icon, it displayed “New Schematic” which gave me reassurance that I was going in the right direction.

# Placing My First Component
I eventually selected a voltage source and placed it onto the schematic. This was easy since I could clearly see a toolbar with multiple basic circuit components. However, after placing it, I noticed that my cursor was still holding another voltage source, and every click attempted to place more.
 
I did not realize that I was still in placement mode. This highlights poor feedback, since the system did not clearly indicate that I was still going to place voltage sources after my first one. It also created confusion due to the fact that I did not know how to exit and get rid of the voltage source in my cursor. Trying to move the mouse around towards the toolbar also made the schematic move as if I was going up and made my previous component disappear from my view.
# Trial and Error to Regain Control
To fix this, I began experimenting and right clicked, which removed the voltage source from my cursor. This allowed me to move freely again and return to my original component.
This moment demonstrates poor recognition rather than recall, since there was no indication that right-clicking would cancel the action. I had to rely on trial-and-error rather than guidance from the system. After inquiring more I also realized that hitting the esc button also canceled the action.
# Navigation and Zoom Issues
Next, I tried to navigate around the workspace. I used the zoom tool, but it zoomed out too much, making my circuit difficult to see.
 
When I tried to use my touchpad, I found that the zoom direction was opposite of what I expected. This created confusion due to a mismatch in my mental model, which refers to how a user expects a system to behave based on prior experience. Since most applications use similar zoom behavior, this made LTspice harder to use and I often found myself getting it wrong over and over again.
Helpful Guidance While Placing Components
When placing a resistor, I noticed a message at the bottom of the screen that said “Ctrl + R to rotate.”
 
This was an example of good feedback, because it provided clear and immediate guidance at the exact moment I needed it. This improved the learnability of the system, since I could quickly understand how to rotate the component without guessing.
# Issues Moving Components
After placing both components, I used the wire tool to connect them and added a ground node to complete the circuit. I then wanted to center the resistor using the move tool, but this caused the wires to disconnect from the component altogether.
 
This did not match my expectations. My mental model was that moving a component in a circuit with wires should keep the wires intact. Instead, the system treated wires and components separately, making the interaction error prone.
# Discovering the Correct Tool
After some trial and error, I discovered the “stretch” tool, which allowed me to move the resistor while keeping the wires connected.
 
Although this tool worked as expected, it was not clear what its purpose was compared to the move tool that was near it. This again reflects poor recognition rather than recall, since an important function was hidden and not clearly explained.
# Assigning Values
Next, I right clicked on the resistor and voltage source to assign values. This opened clear and intuitive menus where I could input values like 5V and 500Ω.
  
This interaction demonstrated strong affordance, since right-clicking to edit properties is a common and familiar action. It also provided good feedback, as the system created popup windows that clearly displayed editable fields.
Final Outcome
Finally, I ran the simulation and plotted the voltage and current values, confirming that my circuit was working correctly.
 
This marked the successful completion of my task.
